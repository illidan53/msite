动用superpowers brainstorm，我想写一个实时跟踪股票价格的网站：
1. 可以自由配置watchlist，watchlist从配置文件动态载入
2. 每个watchlist有类似grafana的panel和对应的row设计，可以展开收回，当前不在展开状态的板块数据就不同步了
3. 数据来源是pologon.io，api key的话从zsh shell的POLYGON_API_KEY环境变量获取，如果后续要把这个变量存到github actions上，从环境变量写到环境变量，避免打印明文也避免把这个明文发给大模型
4. 归根结底，每个watchlist可能对应一个特定的板块（例如半导体、科技股等），我希望其中显示市场热点关注的、体量最大的龙头以及用户自己关注的其中的股票。但是不要加起来显示过于臃肿，也要优先显示用户重点关注的；watchlist也可以是用户自己乱定义的。所以在创建watchlist的时候可以跳出特定的提示问用户是不是要构建特定的板块
5. 参考本机上的~/projects/cgame项目，将这个网页利用容器化 + pulumi + github actions部署到nphunter.net的域名下（本机aws sso认证route53，finance.nphunter.net，infra代码落地在这个repo里，但需要在~/projects/nphunter-site里的主页加一个导向）
6. 页面上提供可以切换view的能力，可以有表格，time series，页面上面可以像普通股票软件设计一样，添加各种时间轴选项
7. 研究一下股票的波浪理论，代码里设计数理模型，先检查所有数据，然后开始找出所有可疑区间，询问一浪大区间是不是这段，如果不是继续下一个可能性，问完大浪区间后询问这个大浪区间是否已经结束、是否关心一浪中的小浪区间进行进一步识别；继续下一个直到已经到了数据最后或者某个时间点等。模块有能力交互提示预测新浪范围等等，需要brainstorm一下
8. 页面上可以配置刷新数据的时间频率，查阅polygon免费版和订阅版能承载的api rate limit，基于当前的板块数据量动态将过高的刷新频率加灰，太接近极限的也给予一个提醒的颜色
9. 用react框架，提供比较完备的交互设计和模块选择，强调交互体验和设计
